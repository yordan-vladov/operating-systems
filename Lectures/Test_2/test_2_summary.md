## Линкове

### Какво трябва да знаете

- Линкът е системен ресурс, който реферира към друг ресурс (файл или директория).
- Има два вида линкове: **твърд** и **мек (символен)**.

---

### Твърд линк

- Файл, който сочи към **същите данни (inode)** като оригиналния файл.
- Изтриването на оригинала **не унищожава данните** — те се пазят, докато всички твърди линкове не бъдат изтрити.
- Работи **само в същата файлова система**.
- Може да сочи **само към файлове**, не към директории.

```bash
ln file.txt hard_link.txt
```

---

### Мек (символен) линк

- Файл, който съдържа **пътя** до друг ресурс.
- Ако оригиналният файл бъде изтрит → линкът се **счупва (broken link)**.
- Може да сочи към **файлове и директории**.
- Работи **между различни файлови системи**.

```bash
ln -s file.txt soft_link.txt
```

---

### Сравнение

| Характеристика        | Твърд линк       | Мек линк             |
| --------------------- | ---------------- | -------------------- |
| Споделя inode         | Да               | Не                   |
| Работи без оригинала  | Да               | Не (broken link)     |
| Към директории        | Не               | Да                   |
| Между файлови системи | Не               | Да                   |

---

### Нужни команди

```bash
ln file link              # твърд линк
ln -s file link           # мек линк
ls -li                    # показва inode номера
find ~ -type l            # намира меки линкове
find ~ -xtype l           # намира счупени меки линкове
```

---

### Примерна задача — твърд линк

Напишете команди, които:
- Създават файл `data.txt`
- Създават твърда връзка `hard.txt`
- Показват inode номерата
- Изтриват оригиналния файл
- Проверяват съдържанието на връзката

---

```bash
echo "Hello" > data.txt
ln data.txt hard.txt
ls -li data.txt hard.txt
rm data.txt
cat hard.txt              # Все още работи!
```

---

### Примерна задача — мек линк

Напишете команди, които:
- Създават директория `data`
- Създават символна връзка `data_link`, сочеща към `data`
- Създават файл `notes.txt` вътре в `data` чрез връзката
- Проверяват съдържанието на директорията `data`

---

```bash
mkdir data
ln -s data data_link
touch data_link/notes.txt
ls data
```

## Shell Скриптове

### Какво трябва да знаете

- Структура на скрипт: shebang + команди.
- Работа с променливи, аритметика, условия, цикли.
- Четене на аргументи от командния ред.

---

### Структура на скрипт

```bash
#!/bin/bash

# Коментар
echo "Hello, World!"
```

Изпълнение:
```bash
chmod +x script.sh
./script.sh
```

---

### Променливи и аритметика

```bash
NAME="Linux"
echo $NAME

NUM=10
echo $((NUM * 2 + 3))   # 23
```

---

### Специални променливи

| Променлива | Значение                     |
| ---------- | ---------------------------- |
| `$0`       | Името на скрипта             |
| `$1`, `$2` | Позиционни аргументи         |
| `$#`       | Брой аргументи               |
| `$@`       | Всички аргументи (разделени) |

---

### Условия

```bash
if [ "$1" -gt 10 ]; then
    echo "По-голямо от 10"
elif [ "$1" -eq 10 ]; then
    echo "Равно на 10"
else
    echo "По-малко от 10"
fi
```

---

### Аритметични оператори за сравнение

| Оператор | Значение            |
| -------- | ------------------- |
| `-eq`    | Равно               |
| `-ne`    | Неравно             |
| `-gt`    | По-голямо           |
| `-lt`    | По-малко            |
| `-ge`    | По-голямо или равно |
| `-le`    | По-малко или равно  |

---

### Файлови оператори

| Оператор | Значение                |
| -------- | ----------------------- |
| `-f`     | Файлът съществува       |
| `-d`     | Директорията съществува |
| `-e`     | Ресурсът съществува     |
| `-x`     | Ресурсът е изпълним     |

```bash
if [ -f "file.txt" ]; then
    echo "Файлът съществува"
fi
```

---

### Примерна задача — file_info.sh

Създайте скрипт `file_info.sh`, който:
- Приема име на файл като аргумент
- Проверява дали файлът съществува
- Ако съществува → извежда размер, inode номер и права
- Ако не съществува → извежда съобщение за грешка

---

```bash
#!/bin/bash

if [ $# -ne 1 ]; then
    echo "Употреба: $0 <файл>"
    exit 1
fi

if [ -f "$1" ]; then
    echo "Размер: $(du -h "$1" | cut -f1)"
    echo "Inode: $(ls -i "$1" | cut -d' ' -f1)"
    echo "Права: $(ls -l "$1" | cut -d' ' -f1)"
else
    echo "Грешка: файлът '$1' не съществува"
fi
```

---

### Примерна задача — backup_logs.sh

Създайте скрипт `backup_logs.sh`, който:
- Създава архив `backup_ДАТА.tar.gz` с текущата дата
- Архивира всички `.log` файлове
- Проверява дали архивът е създаден успешно
- Изтрива оригиналните `.log` файлове само при успех

---

```bash
#!/bin/bash

DATE=$(date +"%Y-%m-%d")
ARCHIVE="backup_${DATE}.tar.gz"

tar -czvf "$ARCHIVE" *.log

if [ -f "$ARCHIVE" ]; then
    echo "Архивът е създаден успешно: $ARCHIVE"
    rm *.log
else
    echo "Грешка: архивът не беше създаден"
fi
```

## Логване

### Какво трябва да знаете

- Логването записва системни събития, грешки и действия.
- Лог файловете се намират в `/var/log`.
- Основни инструменти: `journalctl`, `tail`, `logger`.

---

### Основни лог файлове

| Файл                | Съдържание                           |
| ------------------- | ------------------------------------ |
| `/var/log/syslog`   | Общи системни съобщения              |
| `/var/log/auth.log` | Информация за логване и автентикация |
| `/var/log/kern.log` | Съобщения от ядрото                  |

---

### Нива на лог съобщения

`DEBUG` → `INFO` → `WARNING` → `ERROR` → `CRITICAL`

---

### Преглед на логове

```bash
cat /var/log/syslog
tail -f /var/log/syslog          # в реално време
```

---

### journalctl

```bash
journalctl                       # всички логове
journalctl -b                    # от текущото стартиране
journalctl -u nginx              # логове на услуга
journalctl -u nginx -f           # в реално време
journalctl -p err                # само грешки
journalctl -n 50                 # последните 50 реда
journalctl --since "1 hour ago"  # от преди час
```

---

### Ръчно логване с logger

```bash
logger "Съобщение"
logger -t MY_APP "Стартиране"
logger -p user.err "Грешка"

# Филтриране по таг
journalctl -t MY_APP
```

---

### Примерна задача — system_log.sh

Създайте скрипт `system_log.sh`, който:
- Записва начално съобщение в системния лог с таг `SYSTEM_LOG` чрез `logger`
- Записва текущата дата, потребител и памет в `report.txt`
- Записва крайно съобщение чрез `logger`

```bash
#!/bin/bash

logger -t SYSTEM_LOG "Скриптът стартира"

{
    echo "Дата: $(date)"
    echo "Потребител: $USER"
    echo "Памет:"
    free -h
} > report.txt

logger -t SYSTEM_LOG "Скриптът завърши"
echo "Готово. Вижте report.txt"
```

## Устройства

### Какво трябва да знаете

- В Linux всичко е файл, включително устройствата.
- Устройствата се намират в `/dev`.
- Има два вида: **блокови** и **символни**.

---

### Видове устройства

| Тип      | Буква в `ls -l` | Начин на работа | Пример               |
| -------- | --------------- | --------------- | -------------------- |
| Блоково  | `b`             | На блокове      | `/dev/sda` (диск)    |
| Символно | `c`             | Потоково        | `/dev/tty` (терминал)|

---

### Специални устройства

| Устройство    | Функция                 |
| ------------- | ----------------------- |
| `/dev/null`   | Изхвърля данни          |
| `/dev/zero`   | Генерира нули           |
| `/dev/random` | Генерира случайни числа |

---

### Основни файлови системи

| Файлова система | Характеристика                         |
| --------------- | -------------------------------------- |
| `ext4`          | Най-използвана в Linux, с журнализация |
| `ext2`          | Без журнализация, за флаш устройства   |
| `FAT32`         | Съвместима с всички ОС, файл до 4 GB  |
| `NTFS`          | Windows файлова система                |
| `exFAT`         | За флаш памети, без ограничение 4 GB  |

---

### Нужни команди

```bash
lsblk                        # показва дисковете
lsblk -f                     # показва файловите системи
df -h                        # използвано дисково пространство
sudo fdisk -l                # информация за дяловете

sudo mount /dev/sdb1 /mnt    # монтиране
sudo umount /mnt             # демонтиране

sudo mkfs.ext4 /dev/sdb1     # форматиране с ext4
sudo mkfs.vfat /dev/sdb1     # форматиране с FAT32
```

## Cron задачи

### Какво трябва да знаете

- Cron е системна услуга за автоматично изпълнение на задачи по разписание.
- Задачите се управляват чрез `crontab`.

---

### Формат на cron израз

```
* * * * * команда
│ │ │ │ │
│ │ │ │ └── Ден от седмицата (0–7, 0 и 7 = неделя)
│ │ │ └──── Месец (1–12)
│ │ └────── Ден от месеца (1–31)
│ └──────── Час (0–23)
└────────── Минута (0–59)
```

---

### Специални символи

| Символ | Значение            |
| ------ | ------------------- |
| `*`    | Всяка стойност      |
| `,`    | Изброяване (1,5,10) |
| `-`    | Диапазон (1-5)      |
| `*/n`  | На всеки n          |

---

### Примери за cron изрази

```bash
0 3 * * *       # Всеки ден в 3:00
*/5 * * * *     # На всеки 5 минути
30 8 * * 1      # Всеки понеделник в 8:30
0 0 1 * *       # На 1-во число от всеки месец в полунощ
```

---

### Нужни команди

```bash
crontab -e      # Редактиране на crontab
crontab -l      # Преглед на задачите
crontab -r      # Изтриване на всички задачи
```

---

### Добра практика — пренасочване на изход

```bash
0 3 * * * /home/user/backup.sh >> /home/user/backup.log 2>&1
```

---

### Примерна задача

Напишете команди за:
- Добавяне на cron задача, която изпълнява `/home/user/cleanup.sh` всеки ден в полунощ
- Преглед на всички текущи cron задачи
- Изтриване на всички cron задачи

---

```bash
# Добавяне на задача (отваря редактора — добавете реда):
crontab -e
# Въведете: 0 0 * * * /home/user/cleanup.sh

# Преглед
crontab -l

# Изтриване на всички
crontab -r
```

## Подготовка за теста

### Затворени въпроси — на какво да обърнете внимание

- **Линкове:** Твърд линк оцелява след изтриване на оригинала. Мек линк се счупва.
- **Shell:** `$((израз))` за аритметика; `[ -f ]`, `[ -d ]` за проверки.
- **Cron:** Редът на полетата: минута → час → ден от месец → месец → ден от седмица.
- **journalctl:** `-f` = в реално време; `-u` = конкретна услуга; `-p err` = само грешки.
- **Устройства:** `b` = блоково, `c` = символно; `df -h` за дисково пространство.

---

### Отворени въпроси — типични скриптове

1. **Архивиране с проверка:** `tar`, `if [ -f архив ]`, `rm` при успех
2. **Работа с линкове:** `ln`, `ln -s`, `ls -li`, `cat`
3. **Проверка на файл:** `if [ $# -ne 1 ]`, `if [ -f "$1" ]`, `ls -l`, `ls -i`
4. **Cron задача:** `crontab -e` + правилен израз
5. **Системен лог скрипт:** `logger -t`, `date`, `free -h`, пренасочване в файл
