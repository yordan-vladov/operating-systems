## Какво е CRON?

* **CRON** е системна услуга в Linux за автоматично изпълнение на задачи.
* Работи във фонов режим (daemon).
* Позволява планиране на команди или скриптове по време.

---

## Какво е cron задача?

* Команда или скрипт, който се изпълнява автоматично.
* Изпълнява се по предварително зададен график.
---
* Използва се за:
    * Архивиране
    * Почистване на временни файлове
    * Изпращане на отчети
    * Стартиране на скриптове

---

## Формат на cron израз

```bash
* * * * * command
│ │ │ │ │
│ │ │ │ └── Ден от седмицата (0–7) (0 и 7 = Неделя)
│ │ │ └──── Месец (1–12)
│ │ └────── Ден от месеца (1–31)
│ └──────── Час (0–23)
└────────── Минута (0–59)
```

---

### Примери за стойности

| Символ | Значение            |
| ------ | ------------------- |
| `*`    | всяка стойност      |
| `,`    | изброяване (1,5,10) |
| `-`    | диапазон (1-5)      |
| `*/n`  | на всеки n          |

## Създаване на cron задача

---

### Редактиране на личния crontab файл:

```bash
crontab -e
```

---

### Преглед на текущите задачи:

```bash
crontab -l
```

---

### Изтриване на всички задачи:

```bash
crontab -r
```

---

## Примери

### Изпълнение на скрипт всеки ден в 3:00 сутринта

```bash
0 3 * * * /home/user/backup.sh
```

---

### Изпълнение на команда на всеки 5 минути

```bash
*/5 * * * * /home/user/script.sh
```

---

### Изпълнение всеки понеделник в 8:30

```bash
30 8 * * 1 /home/user/report.sh
```

---

## Системен crontab

Освен потребителския crontab има и системен:

```bash
/etc/crontab
```

---

Форматът е различен (включва потребител):

```bash
* * * * * user command
```

---

## Практически пример

* Имаме уеб сървър.
* Искаме всеки ден да се създава архив на базата данни.
* Създаваме скрипт `backup.sh`
* Добавяме го в crontab:

```bash
0 2 * * * /home/user/backup.sh
```

---

## Логове на CRON

CRON записва логове в:

```bash
/var/log/syslog
```

---

или

```bash
/var/log/cron
```

(зависи от дистрибуцията)

---

## Проверка дали cron работи

```bash
systemctl status cron
```

---

или


```bash
systemctl status crond
```

---

## Важни особености

* Пътищата трябва да са **абсолютни**
* Средата (environment) е ограничена
* Добра практика е да се пренасочва изход:

```bash
0 3 * * * /home/user/script.sh >> /home/user/output.log 2>&1
```

---

## Обобщение

* CRON автоматизира задачи
* Използва се чрез `crontab`
* Форматът има 5 времеви полета
* Подходящ за системна администрация и автоматизация
